# CHAMMI Evaluation

## Overview
This code evaluates foundation models using CHAMMI scores. The implementation is based on [MorphEm](https://github.com/broadinstitute/MorphEm).

---

## Feature Extraction for CHAMMI Scores

### Command
```bash
python feature_extraction.py \
  --root_dir /scr/vidit/Foundation_Models/CHAMMI \
  --feat_dir /scr/vidit/Foundation_Models/features_hpa \
  --model vit \
  --gpu 1 \
  --batch_size 16
```

### Flag Documentation

- `--root_dir`: Path to the root directory containing the CHAMMI dataset. This directory should contain all the image data and metadata required for feature extraction.

- `--feat_dir`: Output directory where extracted features will be saved. The script will create feature files in this location for later use in benchmark evaluation. Create this directory beforehand if it doesn't exist.

- `--model`: Type of model architecture to use for feature extraction (e.g., `vit` for Vision Transformer). Specifies which pretrained model backbone to use.

- `--gpu`: GPU device ID to use for computation (e.g., `0`, `1`, `2`, etc.). Use a single GPU ID for single-GPU processing. Ensure the specified GPU is available on your system.

- `--batch_size`: Number of images to process per batch during feature extraction (typical values: 8, 16, 32, 64). Adjust based on available GPU memory—reduce if you encounter out-of-memory errors.

### Output
The script generates feature files extracted from the model. These files are saved in `--feat_dir` and used by the benchmark evaluation script.

## Evaluating CHAMMI Score

### Command
```bash
python -c "from morphem.benchmark import run_benchmark; run_benchmark('Dataset path', 'Output Directory', 'Features directory', 'pretrained_vit_features.npy')"
```

### Parameter Documentation

The `run_benchmark()` function takes four arguments (in order):

1. **Dataset path**:
   - Path to the root directory of the CHAMMI dataset
   - Must contain the images and metadata for evaluation

2. **Output directory**:
   - Directory where CHAMMI benchmark scores and results will be saved
   - Create this directory beforehand if it doesn't exist

3. **Features directory**:
   - Path to the directory containing extracted features from the feature extraction step
   - Must match the `--feat_dir` used in `feature_extraction.py`
   - The script expects to find feature files in this location

4. **Feature filename**:
   - Name of the feature file to load from the features directory
   - Typically generated by the feature extraction script
   - Should be in NumPy format (`.npy`)

## Workflow Example

A typical workflow would look like:

```bash
# Step 1: Extract features from CHAMMI dataset
python feature_extraction.py \
  --root_dir /scr/vidit/Foundation_Models/CHAMMI \
  --feat_dir /scr/vidit/Foundation_Models/features_Dino_Small_75ds_Guided \
  --model vit \
  --gpu 1 \
  --batch_size 16

# Step 2: Evaluate CHAMMI scores using extracted features
python -c "from morphem.benchmark import run_benchmark; run_benchmark('/scr/vidit/Foundation_Models/CHAMMI', '/scr/vidit/Foundation_Models/CHAMMI_scores/Dino_Small_75ds_Guided', '/scr/vidit/Foundation_Models/features_Dino_Small_75ds_Guided', 'pretrained_vit_features.npy')"
```

### Important Notes
- **Matching paths**: The `--feat_dir` in feature extraction must match the features directory (3rd argument) in `run_benchmark()`
- **Feature filename**: Ensure the feature filename (4th argument) matches the actual filename generated by feature extraction
